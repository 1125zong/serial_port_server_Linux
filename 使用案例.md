# 端口配置工具使用案例

## 案例一：首次使用设备搜索与连接

### 场景描述
用户首次安装并运行端口配置工具，需要搜索局域网内的端口设备并建立连接。

### 操作步骤
1. **启动程序**：双击运行 `chuankou` 可执行文件
2. **搜索设备**：在主界面点击「搜索设备」按钮
3. **查看搜索结果**：程序会自动搜索局域网内的设备，并在表格中显示搜索结果
4. **选择设备**：在搜索结果表格中点击选择要连接的设备
5. **连接设备**：点击「连接」按钮
6. **验证连接**：连接成功后，界面会自动切换到设备管理页面

### 预期结果
- 成功搜索到局域网内的端口设备
- 成功建立与设备的连接
- 界面切换到设备管理页面，可以进行后续配置操作

## 案例二：配置设备网络参数

### 场景描述
用户需要为端口设备配置静态IP地址，以确保设备在局域网内有固定的网络地址。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入网络配置**：在设备管理页面点击「基本设置」按钮
3. **配置LAN1参数**：
   - 选择「静态IP」选项
   - 输入IP地址：192.168.1.100
   - 输入子网掩码：255.255.255.0
   - 输入网关：192.168.1.1
4. **配置LAN2参数**（可选）：
   - 选择「静态IP」选项
   - 输入IP地址：192.168.2.100
   - 输入子网掩码：255.255.255.0
   - 输入网关：192.168.2.1
5. **保存配置**：点击「应用配置」按钮
6. **验证配置**：配置成功后，设备会自动应用新的网络参数

### 预期结果
- 设备成功应用新的网络参数
- 设备可以通过新的IP地址被访问
- 搜索结果表格中显示设备的新IP地址

## 案例三：配置端口参数

### 场景描述
用户需要为设备的端口1配置特定的参数，以满足外部设备的通信需求。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入端口配置**：在设备管理页面点击「端口设置」按钮
3. **选择端口**：在「端口」下拉菜单中选择「端口1」
4. **配置端口参数**：
   - 波特率：9600
   - 数据位：8
   - 停止位：1
   - 校验位：None
   - 流控：None
   - 接口类型：RS-232
5. **保存配置**：点击「应用配置」按钮
6. **验证配置**：配置成功后，设备会自动应用新的端口参数

### 预期结果
- 设备成功应用新的端口参数
- 端口1可以按照新的参数与外部设备通信

## 案例四：设置端口工作模式

### 场景描述
用户需要将设备的端口2设置为TCP Server模式，以允许远程设备通过网络访问该端口。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入工作模式配置**：在设备管理页面点击「模式设置」按钮
3. **选择端口**：在「端口」下拉菜单中选择「端口2」
4. **选择工作模式**：在「工作模式」下拉菜单中选择「TCP Server Mode」
5. **配置TCP参数**：
   - TCP活动检测时间：30秒
   - TCP最大连接数：5
6. **保存配置**：点击「应用配置」按钮
7. **验证配置**：配置成功后，设备会自动应用新的工作模式

### 预期结果
- 设备成功应用新的工作模式
- 端口2以TCP Server模式运行，可以接受远程连接

## 案例五：端口镜像配置

### 场景描述
用户需要将端口1的数据流镜像到端口2，以便通过端口2监控端口1的通信内容。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入端口管理**：在设备管理页面点击「端口管理」按钮，选择「端口自检/重置/镜像」
3. **进入镜像设置**：在端口管理页面切换到「镜像设置」选项卡
4. **启用端口镜像**：勾选「启用端口镜像」选项
5. **选择源端口**：在「源端口」下拉菜单中选择「端口1」
6. **选择目标端口**：在「目标端口」下拉菜单中选择「端口2」
7. **保存配置**：点击「应用配置」按钮
8. **验证配置**：配置成功后，端口1的数据流会被镜像到端口2

### 预期结果
- 设备成功应用端口镜像配置
- 端口1的数据流被实时镜像到端口2
- 可以通过端口2监控端口1的通信内容

## 案例六：设备锁定与解锁

### 场景描述
用户需要锁定设备以防止未授权访问，一段时间后需要解锁设备进行维护。

### 操作步骤
#### 锁定设备
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入设备管理**：在设备管理页面点击「设备管理」按钮，选择「设备操作」
3. **锁定设备**：点击「锁定设备」按钮
4. **确认锁定**：在弹出的确认对话框中点击「确定」
5. **记录解锁码**：系统会生成并显示解锁码，请妥善保管

#### 解锁设备
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入设备管理**：在设备管理页面点击「设备管理」按钮，选择「设备操作」
3. **输入解锁码**：在「解锁码」输入框中输入之前记录的解锁码
4. **解锁设备**：点击「解锁设备」按钮
5. **验证解锁**：设备成功解锁后，可以正常进行配置操作

### 预期结果
- 设备成功锁定，防止未授权访问
- 设备成功解锁，可以进行正常的配置操作

## 案例七：固件升级

### 场景描述
用户需要为设备升级固件，以获取新功能或修复已知问题。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入设备管理**：在设备管理页面点击「设备管理」按钮，选择「设备操作」
3. **选择固件文件**：点击「浏览」按钮，选择要升级的固件文件（.pkg格式）
4. **开始升级**：点击「开始升级」按钮
5. **确认升级**：在弹出的确认对话框中点击「确定」
6. **等待升级完成**：升级过程中不要断开连接或关闭程序
7. **验证升级**：升级完成后，设备会自动重启，重启后可以查看新的固件版本

### 预期结果
- 设备成功升级到新的固件版本
- 设备正常重启并运行
- 设备固件版本显示为新的版本号

## 案例八：恢复出厂设置

### 场景描述
设备配置出现问题，用户需要将设备恢复到出厂默认设置。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入设备管理**：在设备管理页面点击「设备管理」按钮，选择「设备操作」
3. **恢复出厂设置**：点击「恢复出厂设置」按钮
4. **确认操作**：在弹出的确认对话框中点击「确定」
5. **输入确认码**：在弹出的输入对话框中输入「ERASE」以确认操作
6. **等待恢复完成**：设备会自动恢复出厂设置并重启
7. **重新配置**：设备重启后，需要重新进行网络配置等操作

### 预期结果
- 设备成功恢复到出厂默认设置
- 设备自动重启
- 所有之前的配置被清除，需要重新配置

## 案例九：查看端口状态信息

### 场景描述
用户需要查看设备各端口的连接状态和数据传输情况。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入端口管理**：在设备管理页面点击「端口管理」按钮，选择「端口状态信息」
3. **查看状态信息**：系统会显示各端口的状态信息，包括：
   - 端口号
   - 发送计数（TxCnt）
   - 接收计数（RxCnt）
   - 总发送量（TxTotal）
   - 总接收量（RxTotal）
4. **分析数据**：根据显示的状态信息分析端口的通信情况

### 预期结果
- 成功查看各端口的状态信息
- 了解各端口的通信情况和数据传输量

## 案例十：设置看门狗

### 场景描述
用户需要启用设备的看门狗功能，以提高设备的稳定性和可靠性。

### 操作步骤
1. **连接设备**：按照案例一的步骤连接到目标设备
2. **进入设备管理**：在设备管理页面点击「设备管理」按钮，选择「设备操作」
3. **启用看门狗**：勾选「启用看门狗」选项
4. **设置超时时间**：在「超时时间」输入框中设置看门狗超时时间（例如：60秒）
5. **保存配置**：点击「应用配置」按钮
6. **验证配置**：配置成功后，设备会启用看门狗功能

### 预期结果
- 设备成功启用看门狗功能
- 当设备无响应时间超过设定的超时时间时，设备会自动重启
- 提高设备的稳定性和可靠性
