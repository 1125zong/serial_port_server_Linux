# NPort 配置管理 - 快速开始

## 🚀 5分钟上手指南

### 1. 编译项目

```bash
cd QT
qmake
make
```

### 2. 运行程序

```bash
./chuankou
```

### 3. 打开配置管理

点击菜单栏：**配置管理** → **NPort 配置**

### 4. 基本操作

#### 查看当前配置
1. 点击工具栏 **"刷新"** 按钮
2. 树形视图显示所有设备和端口

#### 批量添加端口
1. 点击 **"批量添加"**
2. 输入设备IP地址
3. 选择端口数量 (1/4/8/16/32)
4. 设置起始端口
5. 可选：启用SSL或冗余模式
6. 查看预览
7. 点击 **"确定"**

#### 选择性添加端口
1. 点击 **"选择性添加"**
2. 输入设备IP地址
3. 勾选要添加的端口 (1-32)
4. 设置起始端口
5. 可选：启用SSL或冗余模式
6. 点击 **"确定"**

#### 删除设备
1. 在树形视图中选择设备
2. 点击 **"删除设备"**
3. 确认删除

#### 删除端口
1. 在树形视图中选择一个或多个端口
2. 点击 **"删除端口"**
3. 确认删除

#### 验证配置
1. 点击 **"验证配置"**
2. 查看验证结果

## 📋 配置文件

**位置：** `/etc/wq_nportd.cf`

**格式示例：**
```
ttymajor=34
calloutmajor=39
#[Minor] [ServerIP]  [data]  [cmd]   [FIFO]  [SSL]   [ttyName] [coutName] [interface][mode][BackIP]
0       192.168.1.100   4001    4002    1       0       ttyr00  cur00   0       0
1       192.168.1.100   4003    4004    1       0       ttyr01  cur01   0       0
```

## ⚠️ 注意事项

1. **权限：** 修改配置需要root权限（程序会自动提升）
2. **备份：** 每次修改前自动备份
3. **守护进程：** 修改后自动重新加载

## 🔧 故障排除

### 无法打开配置文件
- 检查文件是否存在：`ls -l /etc/wq_nportd.cf`
- 检查权限：`sudo chmod 644 /etc/wq_nportd.cf`

### 命令执行失败
- 确保命令在PATH中：`which mxaddsvr`
- 检查权限提升工具：`which pkexec` 或 `which sudo`

### 守护进程重新加载失败
- 手动重启：`sudo systemctl restart npreal2d`
