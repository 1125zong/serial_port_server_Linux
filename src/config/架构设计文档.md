# NPort配置管理模块 - 架构设计文档

## 📚 文档目的

本文档详细说明NPort配置管理模块中所有新增文件的作用、设计思路和使用方法，便于理解代码结构和后续开发维护。

## 🏗️ 整体架构


## 📦 模型层 (Model Layer)

模型层负责数据的表示、存储和基本操作。


### 1. PortModel (端口模型)

**文件**: `models/PortModel.h` + `models/PortModel.cpp`

**作用**: 表示单个端口配置条目

**核心功能**:
- 存储端口的所有属性（minor号、IP地址、端口号、TTY名称等）
- 序列化：将对象转换为配置文件格式（tab分隔）
- 反序列化：从配置文件行解析为对象
- 相等性比较

**关键方法**:
```cpp
// 序列化为配置文件行
QString toConfigLine() const;

// 从配置文件行反序列化
static PortModel fromConfigLine(const QString& line, bool* ok = nullptr);

// 相等性比较
bool operator==(const PortModel& other) const;
```

**配置文件格式**:
```
[Minor] [ServerIP] [data] [cmd] [FIFO] [SSL] [ttyName] [coutName] [interface] [mode] [BackIP]
```

**使用场景**:
- 读取/写入配置文件时的数据单元
- 在UI中显示单个端口信息
- 添加/删除端口时的数据载体

---

### 2. DeviceModel (设备模型)

**文件**: `models/DeviceModel.h` + `models/DeviceModel.cpp`

**作用**: 表示一个NPort设备及其所有端口

**核心功能**:
- 管理同一IP地址下的所有端口
- 端口容量推断（1/4/8/16/32端口）
- 端口的增删查操作

**关键方法**:
```cpp
// 添加端口
void addPort(const PortModel& port);

// 删除端口
void removePort(int dataPort, int cmdPort);

// 查找端口
PortModel* findPort(int dataPort, int cmdPort);

// 推断设备容量
int inferredCapacity() const;
```

**容量推断逻辑**:
```
端口数量 → 推断容量
1        → 1
2-4      → 4
5-8      → 8
9-16     → 16
17-32    → 32
```

**使用场景**:
- 按设备分组显示端口
- 批量操作同一设备的端口
- 设备容量管理

---

### 3. ConfigurationData (配置数据)

**文件**: `models/ConfigurationData.h` + `models/ConfigurationData.cpp`

**作用**: 管理整个配置文件的数据

**核心功能**:
- 配置文件的读取和写入
- 设备的增删查操作
- 资源分配（minor号、TTY名称）
- 资源使用检查

**关键方法**:
```cpp
// 文件I/O
bool parseFromFile(const QString& filePath);
bool writeToFile(const QString& filePath);

// 设备管理
void addDevice(const DeviceModel& device);
void removeDevice(const QString& ip);
DeviceModel* findDevice(const QString& ip);

// 资源分配
int getNextAvailableMinor() const;
QString getNextAvailableTty() const;
bool isMinorInUse(int minor) const;
bool isTtyInUse(const QString& ttyName) const;
```

**配置文件路径**: `/etc/wq_nportd.cf`

**使用场景**:
- 应用启动时加载配置
- 配置修改后保存
- 资源冲突检测

## 🎮 控制器层 (Controller Layer)

控制器层负责业务逻辑的协调和流程控制。


### 4. BackupManager (备份管理器)

**文件**: `controllers/BackupManager.h` + `controllers/BackupManager.cpp`

**作用**: 管理配置文件的备份和恢复

**核心功能**:
- 创建带时间戳的备份文件
- 恢复备份文件
- 自动备份路径生成

**关键方法**:
```cpp
// 创建备份
QString createBackup(const QString& sourceFile);

// 恢复备份
bool restoreBackup(const QString& backupFile, const QString& targetFile);
```

**备份文件命名**:
```
原文件: /etc/wq_nportd.cf
备份文件: /etc/wq_nportd.cf.backup.20260115_143022
```

**使用场景**:
- 修改配置前自动备份
- 操作失败时自动恢复
- 手动恢复历史配置

---

### 5. CommandExecutor (命令执行器)

**文件**: `controllers/CommandExecutor.h` + `controllers/CommandExecutor.cpp`

**作用**: 执行系统命令（mxaddsvr、mxdelsvr等）

**核心功能**:
- 执行moxa工具命令
- 自动权限提升（pkexec/sudo）
- 命令执行结果处理

**关键方法**:
```cpp
// 添加端口
bool executeMxaddsvr(const PortModel& port);

// 删除端口
bool executeMxdelsvr(int minor);

// 格式化配置
bool executeMxcfmat();

// 删除TTY节点
bool executeMxrmnod(const QString& ttyName);

// 重载守护进程
bool signalDaemonReload();
```

**权限提升策略**:
1. 优先使用pkexec（图形界面）
2. 降级使用sudo（命令行）
3. 失败时返回错误

**使用场景**:
- 添加端口后执行mxaddsvr
- 删除端口后执行mxdelsvr
- 配置修改后重载守护进程

---

### 6. ConfigManager (配置管理器)

**文件**: `controllers/ConfigManager.h` + `controllers/ConfigManager.cpp`

**作用**: 高级配置操作的协调者

**核心功能**:
- 批量添加端口
- 选择性添加端口
- 删除设备/端口
- 配置验证
- 集成备份和命令执行

**关键方法**:
```cpp
// 加载和保存
bool loadConfiguration();
bool saveConfiguration();

// 批量添加（1/4/8/16/32端口）
bool batchAddPorts(const QString& ip, int portCount, 
                   int startPort, bool ssl, 
                   bool redundant, const QString& backupIp);

// 选择性添加（指定端口索引）
bool selectiveAddPorts(const QString& ip, 
                       const QList<int>& portIndices,
                       int startPort, bool ssl,
                       bool redundant, const QString& backupIp);

// 删除操作
bool deleteDevice(const QString& ip);
bool deleteSelectedPorts(const QString& ip, 
                         const QList<QPair<int,int>>& ports);

// 验证
QStringList validateConfiguration();
```

**操作流程**:
```
1. 创建备份
2. 修改配置数据
3. 保存配置文件
4. 执行系统命令
5. 重载守护进程
6. 失败时恢复备份
```

**使用场景**:
- UI层调用的主要接口
- 所有配置修改操作的入口

## 🖼️ 视图层 (View Layer)

视图层负责用户界面的显示和交互。


### 7. ConfigDialog (主配置对话框)

**文件**: `views/ConfigDialog.h` + `views/ConfigDialog.cpp`

**作用**: 主配置管理界面

**核心功能**:
- 树形视图显示所有设备和端口
- 工具栏提供操作按钮
- 状态栏显示操作结果

**UI组件**:
```cpp
QTreeWidget* m_treeWidget;     // 树形视图（9列）
QToolBar* m_toolbar;           // 工具栏
QStatusBar* m_statusBar;       // 状态栏
ConfigManager* m_configManager; // 配置管理器
```

**工具栏按钮**:
1. 刷新 - 重新加载配置
2. 批量添加 - 打开批量添加对话框
3. 选择性添加 - 打开选择性添加对话框
4. 删除设备 - 删除选中的设备
5. 删除端口 - 删除选中的端口
6. 验证配置 - 检查配置有效性

**树形视图列**:
```
IP地址 | Minor | 数据端口 | 命令端口 | FIFO | SSL | TTY | Cout | 模式 | 备份IP
```

**关键方法**:
```cpp
// UI创建
void setupUi();
void createToolbar();
void createTreeWidget();

// 数据填充
void populateTree();

// 操作槽函数
void onRefreshClicked();
void onBatchAddClicked();
void onSelectiveAddClicked();
void onDeleteDeviceClicked();
void onDeletePortsClicked();
void onValidateClicked();
```

**使用场景**:
- 从主窗口菜单打开
- 查看所有配置
- 执行各种配置操作

## 🔧 工具层 (Utility Layer)
